# Context: Internationalization (i18n)

## 1. Overview
Supports multiple languages and cultural conventions (dates, numbers) to make the application accessible to a global audience.

## 2. Technology Stack
- **Library**: i18next + react-i18next
- **Detection**: i18next-browser-languagedetector
- **Backend**: (If needed) HTTP Header `Accept-Language`

## 3. Architecture

### 3.1 Configuration
- **Setup**: `src/i18n/config.ts` initializes the instance.
- **Resources**: Translation files (`locales/en.json`, `locales/es.json`) stored in `public/` or bundled in `src`.
- **Namespaces**: Organize strings by feature (e.g., `common`, `auth`, `projects`).

### 3.2 Usage
- **Hook**: `useTranslation()` for components.
- **Interpolation**: Use `{{value}}` syntax for dynamic values.
- **Ant Design**: Must sync Ant Design's internal locale (via `ConfigProvider`) with `i18n.language`.

### 3.3 Best Practices
- **Keys**: Use nested keys for organization (e.g., `dashboard.welcome_message`).
- **Dates**: Use `dayjs` locale functions for date formatting.
